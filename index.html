Testning

Varför?

  • Verifiera funktionalitet
  • Reproducera buggar
  • Minimera riskerna med att ändra programmet
  • Dokumentera funktionalitet

Vad?

  • Unit test
  • Integration test
  • Property baserad test

Unit

  • En isolerad komponent i systemet
  • Testar specifik funktionalitet
  • Vanligt att dessa inte ändrar state, som i en databas eller DOMen

Integration

  • Tester flera komponenters integration med varandra
  • Testar funktionalitet, över flera komponenter
  • Vanligare att dessa kan ändra state, som i en databas eller DOMen

Property baserad testning

  • Istället för, eller tillsammans med, unit eller integration tester
  • Beskriver en modell för hur "systemet" eller "enheten" ska fungera
  • Genererar tester baserat på modellen som är specifierad

Exempel: Batcha R antal requets på S antal sekunder

  • Detta är en del-feature av ett större projekt

Visa exempel live

Vad är input och output?

  • Input är antal gånger och tid att portionera ut requesten
  • Output är en lista med millisekunder, som då kan användas för att batcha
  • Detta är ett JS system, därför blir outputen i millisekunder, för att lättare passa in med t.ex. timeout()
  • Vår lilla dela av systemet är alltså endast delen som räknar ut när vi ska göra något.
  • Koden som faktiskt gör requesten är på annat ställe.
  • Vår modul går att använda till vilken typ av request som helst, som behöver batchas.
